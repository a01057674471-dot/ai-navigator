# AI 네비게이터 — MVP 기획서

> 내 상황에 맞는 AI를 고르고, AI의 변화를 따라가는 한국어 서비스

## 1. 제품 한 줄 정의

사용자가 하려는 일을 자연어로 입력하면 **작업 목적·품질·가격·한국어·보안** 조건을 기준으로 가장 적합한 AI 도구를 추천하고, 최신 AI 뉴스와 활용 인사이트를 짧고 쉽게 전달하는 반응형 웹앱.

## 2. 초기 타깃

- AI를 업무에 활용하고 싶지만 도구가 너무 많아 선택하기 어려운 직장인
- 콘텐츠를 만드는 크리에이터·1인 사업자
- 리서치·문서·코딩에 AI를 처음 도입하는 학생과 실무자

### 핵심 사용자 문제

1. “이 작업에는 어떤 AI가 맞지?”
2. 서비스마다 가격·무료 한도·지원 기능이 달라 비교하기 어렵다.
3. AI 뉴스가 많지만 내 업무에 어떤 의미가 있는지 알기 어렵다.
4. 추천 결과를 바로 실행할 Prompt와 사용 순서가 부족하다.

## 3. 핵심 가치 제안

- **상황 중심**: “최고의 AI”가 아니라 “지금 내 일에 맞는 AI”를 추천
- **비교 가능**: 적합도, 비용, 한국어, 속도, 보안, 사용 난이도 표준화
- **실행 가능**: 추천 이유와 바로 쓸 수 있는 Prompt 제공
- **검증 가능**: 정보 기준일, 공식 출처, 업데이트 이력 표시
- **한국어 우선**: 국내 사용자가 이해하기 쉬운 설명과 사용 사례

## 4. MVP 범위

### 포함

- 반응형 웹앱(모바일 우선)
- 홈 대시보드
- 자연어 작업 입력 → 상황별 추천 결과
- AI 도구 디렉터리와 카테고리 필터
- 도구 상세 비교 진입점
- AI 뉴스·인사이트 카드
- 상황별 추천 바로가기
- 관심 목록·Prompt 저장을 위한 UI 진입점
- 관리자 등록을 고려한 콘텐츠 데이터 구조

### 후순위

- AI 서비스 API를 앱 안에서 직접 실행
- 실시간 벤치마크 자동화
- 사용자 리뷰·커뮤니티
- 네이티브 iOS/Android 앱
- 유료 구독 및 팀 기능

## 5. 정보 구조

- 홈
  - 오늘의 탐색
  - AI 추천받기
  - AI 도구 탐색
  - AI 뉴스·인사이트
- 내 공간
  - 저장한 AI / Prompt
  - 관심 분야 설정
- 향후
  - AI 상세 페이지
  - 비교하기
  - 뉴스 상세 페이지
  - 개인 맞춤 브리핑

## 6. 홈 화면 구성

1. 상단: 날짜, 알림, 프로필
2. Hero: “지금 하려는 일, 어떤 AI가 가장 잘할까요?” 입력창
3. 추천 카드: 적합도·추천 이유·태그·가격
4. 인사이트 배너: 성능 외 가격·보안·한국어도 비교하라는 메시지
5. 도구 탐색: 글쓰기, 이미지, 영상, 리서치, 코딩 필터
6. 오늘의 브리핑: 출시·워크플로우·인사이트 뉴스
7. 상황별 바로가기: 회의록, 콘텐츠, PDF, 코딩 오류

## 7. 추천 로직 설계

### MVP의 2단 구조

```text
사용자 자연어 입력
  → 작업 카테고리 분류
  → 구조화된 AI 도구 데이터 필터링
  → 사용자 조건별 점수 계산
  → LLM이 추천 이유·사용법·Prompt를 설명
```

### 도구 평가 필드

```json
{
  "name": "도구 이름",
  "category": ["writing", "research"],
  "best_for": ["회의록", "긴 문서", "리서치"],
  "korean_score": 5,
  "quality_score": 5,
  "speed_score": 4,
  "privacy_score": 3,
  "difficulty_score": 2,
  "price_type": "freemium",
  "free_limit": "무료 플랜 설명",
  "last_verified_at": "2026-07-20",
  "official_url": "https://..."
}
```

### 추천 점수 예시

```text
총점 = 작업 적합도 40%
     + 품질 20%
     + 비용 적합도 15%
     + 한국어 10%
     + 속도 5%
     + 보안 10%
```

사용자가 “무료 우선”, “보안 우선”을 선택하면 가중치를 변경한다.

## 8. 뉴스 운영 원칙

- 공식 블로그·제품 페이지·신뢰 가능한 매체를 우선 수집
- 원문을 복사하지 않고 자체 요약·해설 작성
- 제목, 3줄 요약, 왜 중요한가, 누구에게 유용한가, 바로 해볼 일, 출처를 저장
- 모든 콘텐츠에 발행일과 정보 기준일 표시
- 가격·정책·모델 성능은 변경 가능성을 표시
- 뉴스 자동 생성 후 관리자 검수 및 발행

## 9. 데이터 모델 초안

### `tools`

- `id`, `name`, `maker`, `description`, `official_url`
- `categories[]`, `best_for[]`, `strengths[]`, `weaknesses[]`
- `quality_score`, `korean_score`, `speed_score`, `privacy_score`
- `price_type`, `pricing_summary`, `free_limit`
- `last_verified_at`, `status`

### `news`

- `id`, `title`, `summary`, `insight`, `action_tip`
- `category`, `source_name`, `source_url`
- `published_at`, `verified_at`, `status`

### `prompts`

- `id`, `title`, `task_category`, `body`, `tool_ids[]`
- `visibility`, `created_by`, `created_at`

### `users`

- `id`, ` 관심 분야[]`, `saved_tool_ids[]`, `saved_prompt_ids[]`

## 10. 권장 기술 스택

- Frontend: Next.js + TypeScript
- UI: Tailwind CSS 또는 디자인 토큰 기반 CSS
- Backend/DB: Supabase(PostgreSQL)
- 검색: PostgreSQL Full-text Search → 추후 pgvector
- AI: 작업 분류·추천 설명·뉴스 요약 API
- 수집: RSS/API/허용된 공식 소스
- 배포: Vercel + Supabase
- 운영: 관리자용 CMS 또는 내부 admin route

## 11. 성공 지표

- 추천 검색 후 상세 보기 클릭률
- 검색 → 저장 전환율
- 뉴스 카드 열람률과 저장률
- 추천 결과 만족도(👍/👎 + 이유)
- 7일 재방문율
- 추천 결과에서 공식 도구로 이동하는 비율

## 13. 현재 포함된 데모

`index.html`은 다음을 포함하는 정적 MVP 시안이다.

- AI 네비게이터 브랜드와 반응형 레이아웃
- 검색창과 자연어 작업별 추천 필터
- AI 도구 카테고리 필터
- 뉴스·인사이트·상황별 바로가기
- 모바일 하단 내비게이션
- 상세 기능을 위한 토스트 피드백

실서비스 전환 시 정적 JavaScript 데이터를 API와 DB로 교체하면 된다.
